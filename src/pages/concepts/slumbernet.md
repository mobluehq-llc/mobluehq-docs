slumberNet is the name for MOBLUEHQ's approach to **signed payloads** between your forge instance and network services (for example, a future triage worker for bug reports). Your install holds a keypair. Services you approve can verify that a payload really came from your instance.

You do not need to configure slumberNet to read docs or use chat. It matters when a feature says "signed submit" in release notes.

## Onboarding

If you are joining the volunteer mesh, start with the [slumberNet quickstart](/quickstart/slumbernet).
