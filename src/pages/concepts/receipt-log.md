description: Where to read the history of model and tool activity for a session.
---

The receipt log is the scrollable list attached to a dispatch. Each entry is one step: a model call, a terminal command, an edit, or a policy decision.

When something looks wrong, start with the last receipt that succeeded, then read forward. The log is designed to be shareable: you can paste it into a bug report when you use the in-product Report flow.

For export options, see [Export receipts](/how-to/export-receipts) (stub: wiring varies by build).
