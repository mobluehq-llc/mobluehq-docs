**blueChart** builds **evidence packs** for utilization review: criteria cited, sources quoted verbatim, disagreement shown when models diverge, and every pack **signed** so you can verify offline. The architecture does **not** perform autonomous denials — it produces evidence for a human decision.

Access is **by private demo or customer agreement** today. If you do not have an installer link, contact MOBLUEHQ through your onboarding channel.

## Install

1. Use the blueChart installer or demo URL from your onboarding email.
2. Activate the device on first launch ([licensing](/getting-started/licensing)).
3. Confirm you can reach the pack builder home screen without credential errors.

## First run

1. Start a **new evidence pack** and select or paste the clinical criteria set your organization uses (starter templates ship in demo builds when enabled).
2. Attach or paste the case facts blueChart should cite against (de-identified per your policy).
3. Run the pack. Watch the progress strip — council lanes may run in parallel.
4. Open the finished pack: you should see element-by-element criteria mapping, verbatim quotes, and any **dissent** block when reviewers disagreed.

## Where results and receipts appear

| Output                     | Location                                                                                                                    |
| -------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| Evidence pack (human view) | Pack viewer in the app                                                                                                      |
| Criteria ↔ source mapping  | Per-element sections in the pack                                                                                            |
| Dissent                    | Highlighted section when council split                                                                                      |
| Signed pack / receipt      | **Export pack** or **Download signed JSON** — offline verification per [Offline and privacy](/concepts/offline-and-privacy) |

Store exports in your UM record system. The signature proves the pack contents at export time; re-verify after any copy.

## Limitations (shipped today)

- Private demo — not a self-serve public download yet
- Template libraries vary by customer agreement
- Autonomous approve/deny actions are **not** shipped by design
