**Apprentice** — The agent path that runs tools and edits files. Called out by name so you know when work is not just chat.

**blueForge** — The MOBLUEHQ desktop forge product (lowercase b, camel Forge). The app you run locally.

**blueGlued** — MOBLUEHQ term for the glue layer that keeps editor, chat, and tool policy coherent (spelled with a d at the end).

**Council** — Multi-model review before promoting an answer. Sometimes called council mode in settings.

**Faust Foundation** — The nonprofit MOBLUEHQ works with for governance; mentioned when releases touch foundation policy, not day-to-day chat.

**Linda** — Customer-facing vertical experiences that sit on the same substrate; documentation may call out Linda-specific phrasing.

**MOBLUEHQ** — The organization shipping forge and related services (all caps by convention).

**Receipt** — One recorded step in a dispatch: model or tool, inputs, outputs, and timing.

**slumberNet** — MOBLUEHQ name for signed instance keypairs and verified payloads on the wire.

**Smith** — The chat panel you type into.

**Verification** — The product habit of showing evidence. Also the name of specific UI blocks that compare model outputs.

See also [Smith and Apprentice](/concepts/smith-and-apprentice) and [Report bugs](/how-to/report-bugs).
